Understanding Booleans

Booleans represent logical entities and can only be true or false.

Boolean Operations

Common logical operations include AND (&&), OR (||), and NOT (!). 


      // AND operation
      console.log(true && true); // true
      console.log(true && false); // false
      
      // OR operation
      console.log(true || false); // true
      console.log(false || false); // false

NOT Operator

The NOT operator (!) negates a Boolean value.


      console.log(!true); // false
      console.log(!false); // true

Comparison Operators

Booleans are often the result of comparisons:

  • Greater than: >
  • Less than: <
  • Greater or equal: >=
  • Less or equal: <=
  • Loose equality: ==
  • Strict equality: ===
  • Loose inequality: !=
  • Strict inequality: !==

Truthiness and Falsiness

In JavaScript, values can be "truthy" or "falsy", influencing boolean logic beyond simple true or false. 


      // Truthy examples
      console.log(Boolean(true)); // true
      console.log(Boolean("string")); // true
      
      // Falsy examples
      console.log(Boolean(false)); // false
      console.log(Boolean(0)); // false

Understanding the truthiness or falsiness of different values is crucial for control flow and logic in JavaScript.

Converting to Boolean

JavaScript provides the Boolean function to explicitly convert values to boolean. 


      // Conversion examples
      console.log(Boolean(-2)); // true
      console.log(Boolean(undefined)); // false

Numerical values, strings, arrays, and objects are generally "truthy", whereas 0, "" (empty string), null, undefined, and NaN are "falsy".

Null and Undefined

null is an assignment value that represents the intentional absence of any object value. undefined is a property of the global object, representing the primitive value undefined. 


      let unassigned;
      console.log(unassigned); // undefined
      
      let empty = null;
      console.log(empty); // null

Use null to represent a deliberate non-value and undefined to indicate variables not yet initialized.

Immutability of Primitive Data Types

In JavaScript, primitive data types (like numbers, strings, and Booleans) are immutable. This means that once a primitive value is created, it cannot be altered. 


      let name = "Jane";
      name[0] = "L"; // Attempt to change the first character
      console.log(name); // "Jane"

Variables can be reassigned to new values, but the original value itself cannot be changed.
