Introduction to Node.js

Node.js is a JavaScript runtime built on Chrome's V8 JavaScript engine. It allows you to run JavaScript on the server.

History of Node.js

Created by Ryan Dahl in 2009, Node.js was developed to address the problem of high concurrency with low overhead.

Why Use Node.js?

  • Enables JavaScript to be used for server-side scripting.
  • Non-blocking, event-driven architecture suitable for I/O heavy operations.
  • Large ecosystem of open-source libraries available via npm.

Node Package Manager (npm)

npm is the world's largest Software Registry. It is also a software Package Manager and installer for Node.js.

Why Use npm?

  • Manage project dependencies.
  • Share packages with other developers.
  • Discover reusable code for your projects.

Using Node.js and npm

First, install Node.js from the official website. This will also install npm. 


        // Verify Node.js installation
        node -v
        
        // Verify npm installation
        npm -v

npm Packages

Use npm to install packages needed for your project.


        // Install a package locally
        npm install <package-name>
        
        // Install a package globally
        npm install -g <package-name>

Understanding package.json

The package.json file is a manifest for your project. It contains metadata about the project and lists its dependencies.

node_modules and package-lock.json

  • node_modules: Directory where npm packages are stored.
  • package-lock.json: Automatically generated file; locks down the versions of installed packages to ensure consistency.
