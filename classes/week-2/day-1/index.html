Introduction to Object-Oriented Programming

How to structure your code using the principles of Object-Oriented Programming.

What is Object-Oriented Programming?

Object-Oriented Programming (OOP) is a programming paradigm based on the concept of "objects", which can contain data and code: data in the form of properties, and code, in the form of methods.

Thinking in Objects

Thinking in objects means modeling your software based on real-world entities and their interactions. This approach makes your code more modular, reusable, and understandable.

Creating Objects: Literal Pattern

You can create objects directly in JavaScript using the object literal syntax. This is a simple way to define an object and its properties. 


				const car = {
					make: "Toyota",
					model: "Camry",
					year: 2020
				};

Accessing Object Properties

Object properties can be accessed using dot notation or bracket notation. This allows you to read or modify the properties of an object. 


				console.log(car.make); // Toyota
				console.log(car["model"]); // Camry

Creating method withing an object

To add a method to an object, you have to attach a function to a property/key. Note the this keyword, we'll explore it more later. 


				const person = {
					firstName: "John",
					lastName: "Doe",
					fullName: function() {
							return this.firstName + " " + this.lastName;
					}
				};
				console.log(person.fullName()); // "John Doe"

Understanding the 'this' Keyword

The this keyword in JavaScript refers to the object it belongs to. It allows you to access properties and methods of the object from within its methods. 


				const person = {
					firstName: "John",
					lastName: "Doe",
					fullName: function() {
							return this.firstName + " " + this.lastName;
					}
				};
				console.log(person.fullName()); // John Doe

Summary

Object-Oriented Programming helps organize and structure your JavaScript code. By thinking in terms of objects, creating objects using literals, accessing their properties, and utilizing the this keyword, you can make your code more modular and reusable.
