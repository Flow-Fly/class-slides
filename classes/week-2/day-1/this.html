Understanding the 'this' Keyword

Learn how 'this' behaves in different contexts to write more effective JavaScript.

Global Context

In the global context, 'this' refers to the global object. In a browser, it's 'window'. 


console.log(this === window); // true
var color = 'blue';
console.log(window.color); // 'blue'

Function Context

Inside a function, the value of 'this' depends on how the function is called. 


function showColor() {
console.log(this.color);
}
var color = 'green';
window.showColor(); // 'green'

Object Methods

In object methods, 'this' refers to the object that is calling the method. 


const shirt = {
color: 'blue',
showColor: function() {
  console.log(this.color);
}
};
shirt.showColor(); // 'blue'

Class Context

In classes, 'this' refers to the instance of the class.


class Shirt {
constructor(color) {
  this.color = color;
}
showColor() {
  console.log(this.color);
}
}
const myShirt = new Shirt('red');
myShirt.showColor(); // 'red'

Arrow Functions

Arrow functions do not have their own 'this'. They inherit 'this' from the parent scope. 


const pants = {
color: 'black',
showColor: () => {
  console.log(this.color); // undefined or global color if defined
}
};
pants.showColor();

Summary

Understanding the behavior of 'this' in various contexts is key to mastering JavaScript. Remember, its value can change based on where and how it's called.
