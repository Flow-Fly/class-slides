Introduction to Classes in JavaScript

Learn how to use classes to encapsulate and organize your code effectively.

What is a Class?

A class is a blueprint for creating objects with predefined properties and methods. 


class Book {
  constructor(author, title, pages) {
      this.author = author;
      this.title = title;
      this.pages = pages;
  }
  read(pages) {
      console.log(`Reading ${pages} pages of ${this.title}`);
  }
}

Understanding the Constructor and 'new'

The constructor method is a special method for creating and initializing objects. The 'new' keyword creates an instance of a class. 


const myBook = new Book('JK Rowling', 'Harry Potter', 500);
myBook.read(50);

Inheritance

Inheritance allows a class to inherit properties and methods from another class. 


class Novel extends Book {
  constructor(author, title, pages, genre) {
      super(author, title, pages);
      this.genre = genre;
  }
}

Abstraction

Abstraction means hiding the complex reality while exposing only the necessary parts. 


class Library {
  constructor() {
      this.books = [];
  }
  addBook(book) {
      this.books.push(book);
  }
}

Polymorphism

Polymorphism allows objects of different classes to be treated as objects of a common super class. 


const book = new Book('Author', 'Title', 100);
const novel = new Novel('Author', 'Novel Title', 200, 'Fantasy');
const library = new Library();
library.addBook(book);
library.addBook(novel);

Encapsulation

Encapsulation is the bundling of data and the methods that operate on that data. 


class EncapsulatedBook {
  #pagesRead = 0;
  constructor(title, pages) {
      this.title = title;
      this.totalPages = pages;
  }
  read(pages) {
      this.#pagesRead += pages;
  }
}

Summary

Classes in JavaScript provide a powerful way to encapsulate data and functionality. By understanding and applying principles like inheritance, abstraction, polymorphism, and encapsulation, you can design robust and maintainable code.
