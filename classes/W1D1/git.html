Introduction to Version Control

A system that records changes to a file or set of files over time.

Why Git?

Git supports distributed workflows, enabling collaboration among teams.

Creating a local Repository

git init

Initializes a new local Git repository.

❗️ It will be initialized in your pwd

Checking Status

git status

Displays the state of the working directory and staging area.

Staging Files

git add <filename>

Adds files to the staging area, preparing them for commit.

Committing Changes

git commit -m "Commit message"

Records file snapshots in the version history.

Understanding the Git Log

git log

Displays the commit logs.

Working with GitHub

Creating a Repo on GitHub

Create and clone a repository from GitHub.

Adding Files

git add .

Adds all changed files to the staging area.

Committing Files

git commit -m "Your message"

Commits your staged content as a new commit snapshot.

Pushing to GitHub

git push origin main

Uploads all local branch commits to the remote repository.
