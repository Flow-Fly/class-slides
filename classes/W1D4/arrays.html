Introduction to Arrays

Arrays are used to store multiple values in a single variable, allowing for efficient data management and access.

Creating and Accessing Arrays

Create arrays to store various items and access them using their index. 


        const fridge = ["Tomatoes", "Eggs", "Milk"];
        console.log(fridge[0]); // Accessing the first item

Adding Items to an Array

Use push to add items to the end and unshift to add to the beginning. 


        fridge.push("Cheese"); // Adds to the end
        fridge.unshift("Butter"); // Adds to the beginning

Removing Items from an Array

pop removes the last item, and shift removes the first item. Use splice for specific indices. 


        fridge.pop(); // Removes the last item
        fridge.shift(); // Removes the first item

Converting Strings to Arrays

The .split() method turns a string into an array based on a separator. 


      const ingredients = "apples, oranges, bananas";
      const ingredientsArray = ingredients.split(", ");
      console.log(ingredientsArray); 
      // ["apples", "oranges", "bananas"]

Understanding Mutability

In JavaScript, objects and arrays are mutable by default, meaning their data can be changed after creation. 


      const myArray = [1, 2, 3];
      myArray.push(4);
      console.log(myArray); // [1, 2, 3, 4]

This behavior allows for dynamic data manipulation but requires careful handling to avoid unintended side effects.

Why Does Mutability Happen?

Mutability occurs because when you manipulate an array or object, you're altering its data directly in memory. 


      // Both variables point to the same array in memory
      const originalArray = ['a', 'b', 'c'];
      const newArray = originalArray;
      
      // Modifying newArray affects originalArray
      newArray.push('d');
      console.log(originalArray); // ['a', 'b', 'c', 'd']

This behavior underscores the importance of understanding references in JavaScript to manage and predict changes to your data.

Example: Sorting Without Mutation

Sorting can alter the original array. By working with a copy, we maintain the original array's integrity. 


      // Original array
      const fruits = ["banana", "apple", "orange"];
      
      // Making a shallow copy
      const sortedFruits = [...fruits].sort();
      
      console.log(sortedFruits); // Sorted array
      console.log(fruits); // Original array, unchanged

This approach is especially useful in functional programming or any scenario where data immutability is a key concern.

Why Make Copies of Arrays?

Creating copies helps preserve the original array, allowing us to apply transformations or methods without altering the original data.

Copying Arrays

Copying arrays can be done with the spread operator for a shallow copy or structuredClone for a deep copy. 


        const shallowCopy = [...fridge]; // Shallow copy

Iterating Over Arrays

Use for loops and forEach method to iterate over array items. 


        fridge.forEach(item => console.log(item)); // Using forEach
